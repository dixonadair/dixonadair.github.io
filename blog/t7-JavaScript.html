While there are indeed many differences between JavaScript and Ruby, a lot of those differences are largely syntactic, and not fundamental in terms of the way that we think about structuring logic in the two languages. For instance, one difference is the way hashes are written in Ruby vs. the way that their equivalent (objects) are written in JavaScript. In Ruby, the most conventional way to write a hash (if you were writing the hash all at once from the beginning), is as follows: 

  
    Capitals = { "Georgia" => "Atlanta",
        "Alabama" => "Montgomery",
        "Mississippi" => "Jackson" }

In Javascript, the same hash above would be written as an object as follows: 

  
    var Capitals = {
      "Georgia": "Atlanta",
      "Alabama": "Montgomery",
      "Mississippi": "Jackson"
    }
